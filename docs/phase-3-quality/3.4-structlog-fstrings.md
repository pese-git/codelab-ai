# 3.4 — Унификация structlog: убрать f-strings

**Приоритет:** 🟡 Низкий  
**Оценка:** 2 часа  
**Файлы:** `src/codelab/server/storage/global_policy_storage.py` и другие

---

## Проблема

В ряде мест structlog используется с f-strings:

```python
# ПЛОХО — теряется структурированность:
logger.debug(f"Policy file {self._storage_path} does not exist")
logger.error(f"Corrupted JSON in {self._storage_path}: {e}")
```

Это ломает одно из ключевых преимуществ structlog — машинно-читаемые структурированные логи с раздельными полями. При анализе логов нельзя отфильтровать по `path` или сгруппировать по типу ошибки.

---

## Правило

Все вызовы structlog должны использовать **только** keyword-аргументы для передачи данных:

```python
# ПРАВИЛЬНО:
logger.debug("policy_file_not_found", path=str(self._storage_path))
logger.error("json_parse_error", path=str(self._storage_path), error=str(e))
```

---

## Шаг 1 — Найти все нарушения

```bash
# Найти logger вызовы с f-strings:
grep -rn 'logger\.\(debug\|info\|warning\|error\|exception\)(f"' \
  src/codelab/ --include="*.py"

# Альтернатива через ruff (правило G004):
ruff check src/codelab/ --select G004
```

## Шаг 2 — Добавить правило G в ruff

```toml
# pyproject.toml
[tool.ruff.lint]
select = ["E", "F", "I", "UP", "B", "SIM", "G"]   # добавить G
# G004: Logging statement uses f-string formatting
```

## Шаг 3 — Исправить нарушения в `global_policy_storage.py`

```python
# БЫЛО:
logger.debug(f"Policy file {self._storage_path} does not exist")
logger.debug(f"Policy file {self._storage_path} is empty")
logger.error(f"Corrupted JSON in {self._storage_path}: {e}")

# СТАЛО:
logger.debug("policy_file_not_found", path=str(self._storage_path))
logger.debug("policy_file_empty", path=str(self._storage_path))
logger.error("policy_file_corrupted", path=str(self._storage_path), error=str(e))
```

---

## Шаг 4 — Соглашение по именованию событий

Принять единое соглашение: имена событий в snake_case, описывают **что случилось** (прошедшее время или существительное), а не «что мы делаем»:

| Плохо | Хорошо |
|---|---|
| `"reading file"` | `"file_read_started"` |
| `"file read successfully"` | `"file_read_ok"` |
| `f"error: {e}"` | `"file_read_error", error=str(e)` |
| `"Creating session"` | `"session_creating"` |
