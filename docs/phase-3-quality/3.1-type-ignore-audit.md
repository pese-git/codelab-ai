# 3.1 — Аудит и устранение `# type: ignore`

**Приоритет:** 🟡 Средний  
**Оценка:** 2 дня  
**Область:** весь проект (`src/codelab/`)

---

## Проблема

В кодовой базе 34 вхождения `# type: ignore`. Большинство подавляют реальные архитектурные проблемы, а не баги сторонних библиотек. Молчаливые подавления накапливаются, создавая зоны кода вне статической проверки.

---

## Правило

Каждое `# type: ignore` допустимо только если:
1. Оно обходит подтверждённый баг **внешней библиотеки** (не своего кода).
2. Рядом есть комментарий: что именно игнорируется и почему.
3. Есть ссылка на issue в трекере или версию библиотеки, где баг исправлен.

---

## Шаг 1 — Найти все вхождения

```bash
grep -rn "# type: ignore" src/codelab/ --include="*.py" | sort -t: -k1,1
```

Классифицировать каждое по категориям:

| Категория | Действие |
|---|---|
| Ошибка в своём коде | Исправить код, убрать `ignore` |
| Обход архитектурной проблемы | Решить проблему (см. другие задачи) |
| Баг внешней библиотеки | Оставить с комментарием |
| Непонятно зачем | Исследовать и решить |

## Шаг 2 — Приоритетные случаи для исправления

### `# type: ignore[override]` в `ACPTransportService.listen()`

**Файл:** `src/codelab/client/infrastructure/services/acp_transport_service.py`

```python
# БЫЛО: нарушение LSP — разная сигнатура
async def listen(self) -> AsyncIterator[dict[str, Any]]:  # type: ignore[override]
```

```python
# СТАЛО: привести в соответствие с базовым классом
# В TransportService:
@abstractmethod
async def listen(self) -> AsyncIterator[dict[str, Any]]: ...

# В ACPTransportService — убедиться что сигнатура идентична:
async def listen(self) -> AsyncIterator[dict[str, Any]]:  # без ignore!
    ...
```

### `# type: ignore[call-top-callable]` в `Registration.create()`

Устраняется в задаче 2.8 (миграция на dishka — `di_container.py` удаляется).

### `# type: ignore[arg-type]` в structlog вызовах

```python
# БЫЛО:
self._logger.info("creating_session", host=server_host, port=server_port)  # type: ignore[arg-type]
```

Это ошибка типизации structlog. Корректный обходной приём с комментарием:

```python
# structlog использует **kwargs для structured logging, но аннотирован как PrintLogger.
# Это известное ограничение типизации structlog: https://github.com/hynek/structlog/issues/461
self._logger.info(  # type: ignore[call-arg]  # structlog typing limitation
    "creating_session",
    host=server_host,
    port=server_port,
)
```

## Шаг 3 — Добавить `mypy` или `pyright` в CI

Чтобы новые `# type: ignore` не добавлялись без проверки:

```yaml
# .github/workflows/type-check.yml
- name: Run type checker
  run: |
    cd codelab
    uv run ty check src/codelab/
    # или: uv run mypy src/codelab/ --ignore-missing-imports
```

Добавить в `Makefile`:

```makefile
typecheck:
    cd codelab && ty check src/codelab/

lint:
    cd codelab && ruff check src/codelab/

check: lint typecheck test
```

## Шаг 4 — Включить ruff правило для `# type: ignore` без комментария

```toml
# pyproject.toml
[tool.ruff.lint]
select = ["E", "F", "I", "UP", "B", "SIM", "PGH"]  # добавить PGH
# PGH003: Use specific rule codes when ignoring type issues
# PGH004: Use specific rule codes when ignoring type issues
```

Это заставит писать конкретный код ошибки в `# type: ignore[specific-code]`, а не голое `# type: ignore`.
