# 3.9 — Переименование `TestViewModel` во избежание `PytestCollectionWarning`

**Приоритет:** 🟡 Низкий  
**Оценка:** 30 минут  
**Файл:** `tests/client/test_presentation_base_view_model.py`

---

## Проблема

```
PytestCollectionWarning: cannot collect test class 'TestViewModel'
because it has a __init__ constructor
```

`TestViewModel` — базовый класс для тестов, но pytest пытается собрать его как тестовый класс из-за префикса `Test`. Это не приводит к падению тестов, но создаёт шум в выводе и может скрыть реальные проблемы.

---

## Решение

Переименовать базовый класс, убрав префикс `Test`:

```python
# БЫЛО:
class TestViewModel(BaseViewModel):
    """Конкретная реализация BaseViewModel для тестирования."""

    def __init__(self):
        super().__init__()
        # ...

class TestBaseViewModel(TestViewModel):  # ← настоящий тест
    def test_initial_state(self):
        ...
```

```python
# СТАЛО:
class ConcreteViewModel(BaseViewModel):
    """Конкретная реализация BaseViewModel для тестирования."""

    def __init__(self):
        super().__init__()
        # ...

class TestBaseViewModel(unittest.TestCase):  # ← настоящий тест использует ConcreteViewModel
    def setUp(self):
        self.vm = ConcreteViewModel()

    def test_initial_state(self):
        assert self.vm is not None
```

---

## Проверка

```bash
pytest tests/client/test_presentation_base_view_model.py -v
# Ожидаемый результат: нет PytestCollectionWarning
```
