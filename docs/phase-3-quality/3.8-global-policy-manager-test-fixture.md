# 3.8 — Фикстура сброса `GlobalPolicyManager` в тестах

**Приоритет:** 🟡 Низкий  
**Оценка:** 1 час  
**Файл:** `tests/conftest.py` или `tests/server/conftest.py`

---

## Проблема

`GlobalPolicyManager._instance` — singleton на уровне класса. Если один тест инициализирует его, следующие тесты получат тот же экземпляр. Это приводит к нестабильным тестам, зависящим от порядка выполнения.

---

## Решение

Добавить автоматическую фикстуру сброса singleton перед каждым тестом.

```python
# tests/conftest.py (или tests/server/conftest.py для изоляции):

import pytest
from codelab.server.protocol.handlers.global_policy_manager import GlobalPolicyManager


@pytest.fixture(autouse=True)
def reset_global_policy_manager():
    """Сбрасывает singleton GlobalPolicyManager перед каждым тестом.

    autouse=True — применяется автоматически ко всем тестам в scope.
    """
    GlobalPolicyManager.reset_for_testing()
    yield
    GlobalPolicyManager.reset_for_testing()
```

`reset_for_testing()` должен быть добавлен в задаче 2.6:

```python
# global_policy_manager.py
@classmethod
def reset_for_testing(cls) -> None:
    cls._instance = None
    cls._lock = None
```

## Проверка

```bash
# Запустить тесты несколько раз в разном порядке — результат должен быть идентичным:

pytest tests/server/ -v --randomly-seed=123
pytest tests/server/ -v --randomly-seed=999
```
