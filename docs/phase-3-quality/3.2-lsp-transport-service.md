# 3.2 — Исправление нарушения LSP в `ACPTransportService.listen()`

**Приоритет:** 🟡 Средний  
**Оценка:** 3 часа  
**Файлы:** `src/codelab/client/domain/services.py`, `src/codelab/client/infrastructure/services/acp_transport_service.py`

---

## Проблема

`ACPTransportService.listen()` переопределяет `TransportService.listen()` с несовместимой сигнатурой — нарушение принципа подстановки Лисков (LSP):

```
error[invalid-method-override]: Invalid override of method `listen`
async def listen(self) -> AsyncIterator[dict[str, Any]]:  # type: ignore[override]
```

Клиентский код, работающий с абстракцией `TransportService`, не сможет корректно использовать конкретную реализацию без явного касания её типа.

---

## Шаг 1 — Выяснить текущие сигнатуры

```bash
grep -n "def listen" \
  src/codelab/client/domain/services.py \
  src/codelab/client/infrastructure/services/acp_transport_service.py
```

### Типичная ситуация

```python
# domain/services.py (абстракция):
@abstractmethod
async def listen(self) -> AsyncIterator[dict[str, Any]]:
    """Слушать входящие сообщения с сервера."""
    ...

# acp_transport_service.py (реализация — другая сигнатура):
async def listen(self) -> AsyncIterator[dict[str, Any]]:  # type: ignore[override]
    # Реализация
```

Сигнатуры выглядят одинаково, но `ty` / mypy всё равно жалуется. Скорее всего, причина в разных импортах `AsyncIterator` или в том, что один возвращает тип через `yield` (генератор), а другой — через явный `return`.

---

## Шаг 2 — Унифицировать возвращаемый тип

Генераторная функция с `yield` автоматически является `AsyncGenerator[T, None]`, который совместим с `AsyncIterator[T]`, но не всегда распознаётся type checker'ом корректно. Использовать явную аннотацию `AsyncGenerator`:

```python
# domain/services.py:
from collections.abc import AsyncGenerator
from typing import Any

class TransportService(ABC):
    @abstractmethod
    async def listen(self) -> AsyncGenerator[dict[str, Any], None]:
        """Слушать входящие сообщения с сервера."""
        ...
        yield  # Делает метод генераторной функцией, совместимой с реализацией
```

```python
# acp_transport_service.py:
from collections.abc import AsyncGenerator

class ACPTransportService(TransportService):
    async def listen(self) -> AsyncGenerator[dict[str, Any], None]:
        """Слушает входящие сообщения — единственный receive() через BackgroundReceiveLoop."""
        async for message in self._background_loop:
            yield message
```

## Шаг 3 — Обновить все места использования

```bash

grep -rn "\.listen()" src/codelab/ --include="*.py"
```

Типичное использование — не изменится, т.к. `AsyncGenerator` итерируется так же как `AsyncIterator`:

```python
# Это работает и с AsyncIterator, и с AsyncGenerator:
async for message in transport.listen():
    await process(message)
```

## Тесты

```python
from collections.abc import AsyncGenerator
import pytest
from codelab.client.infrastructure.services.acp_transport_service import ACPTransportService
from codelab.client.domain.services import TransportService


def test_transport_service_implements_interface():
    """ACPTransportService должен полностью реализовывать TransportService."""
    assert issubclass(ACPTransportService, TransportService)


@pytest.mark.asyncio
async def test_listen_returns_async_generator(mock_ws_server):
    """listen() должен возвращать AsyncGenerator."""
    service = ACPTransportService(host="localhost", port=mock_ws_server.port)
    await service.connect()
    gen = service.listen()
    assert hasattr(gen, "__aiter__") and hasattr(gen, "__anext__")
    await service.close()
```
