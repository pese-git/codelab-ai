# 3.5 — Устранение дублирующейся логики в `PermissionManager`

**Приоритет:** 🟡 Низкий  
**Оценка:** 2 часа  
**Файл:** `src/codelab/server/protocol/handlers/permission_manager.py`

---

## Проблема

Методы `should_request_permission()` и `get_remembered_permission()` содержат дублирующую логику чтения `permission_policy`:

```python
def should_request_permission(self, session, tool_kind) -> bool:
    policy = session.permission_policy.get(tool_kind)
    return policy not in {"allow_always", "reject_always"}

def get_remembered_permission(self, session, tool_kind) -> str:
    policy = session.permission_policy.get(tool_kind)
    if policy == "allow_always":
        return "allow"
    if policy == "reject_always":
        return "reject"
    return "ask"
```

При добавлении нового типа политики (например, `"deny_session"`) нужно синхронно обновлять оба метода — о чём легко забыть.

---

## Решение

Ввести единый приватный метод `_resolve_policy()` как источник правды:

```python
from typing import Literal

PolicyDecision = Literal["allow", "reject", "ask"]


class PermissionManager:

    def _resolve_policy(
        self,
        session: SessionState,
        tool_kind: str,
    ) -> PolicyDecision:
        """Единственное место чтения и интерпретации permission_policy.

        Returns:
            "allow"  — инструмент разрешён без запроса
            "reject" — инструмент запрещён без запроса
            "ask"    — нужно запросить у пользователя
        """
        policy = session.permission_policy.get(tool_kind)
        match policy:
            case "allow_always":
                return "allow"
            case "reject_always":
                return "reject"
            case _:
                return "ask"

    def should_request_permission(
        self,
        session: SessionState,
        tool_kind: str,
    ) -> bool:
        """True если нужен интерактивный запрос разрешения."""
        return self._resolve_policy(session, tool_kind) == "ask"

    def get_remembered_permission(
        self,
        session: SessionState,
        tool_kind: str,
    ) -> PolicyDecision:
        """Получить решение из сохранённой политики."""
        return self._resolve_policy(session, tool_kind)
```

При добавлении нового типа политики — изменить только `_resolve_policy()`.

## Тесты

```python
import pytest
from codelab.server.protocol.handlers.permission_manager import PermissionManager
from codelab.server.protocol.state import SessionState


@pytest.fixture
def session():
    return SessionState(session_id="s1", cwd="/", mcp_servers=[])


@pytest.fixture
def manager():
    return PermissionManager()


@pytest.mark.parametrize("policy,expected_ask,expected_decision", [
    ("allow_always",  False, "allow"),
    ("reject_always", False, "reject"),
    (None,            True,  "ask"),
    ("unknown_value", True,  "ask"),
])
def test_policy_resolution(session, manager, policy, expected_ask, expected_decision):
    if policy is not None:
        session.permission_policy["execute"] = policy

    assert manager.should_request_permission(session, "execute") is expected_ask
    assert manager.get_remembered_permission(session, "execute") == expected_decision
```
