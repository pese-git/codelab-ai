# 3.3 — Замена `mcp_manager: Any` на строгий тип

**Приоритет:** 🟡 Низкий  
**Оценка:** 1 час  
**Файл:** `src/codelab/server/protocol/state.py`

---

## Проблема

```python
# state.py
mcp_manager: Any = None
```

`Any` отключает всю статическую типизацию для этого поля. Любое обращение через `session.mcp_manager.some_method()` не проверяется — ошибки в именах методов или типах аргументов обнаружатся только в runtime.

---

## Решение

Использовать `TYPE_CHECKING` для избежания циклических импортов при сохранении строгой типизации.

```python
# БЫЛО (state.py):
from typing import Any

@dataclass(slots=True)  # или class SessionState(BaseModel):
class SessionState:
    ...
    mcp_manager: Any = None
```

```python
# СТАЛО (state.py):
from __future__ import annotations

from typing import TYPE_CHECKING, Any

if TYPE_CHECKING:
    from ..mcp.manager import MCPManager

class SessionState:
    ...
    mcp_manager: MCPManager | None = None  # строгий тип!
```

`TYPE_CHECKING` — это `False` в runtime, поэтому `MCPManager` не импортируется при исполнении (нет циклического импорта). Но type checker видит тип и проверяет все обращения к полю.

---

## Проверка

```bash
# Убедиться что нет циклических импортов:
python -c "from codelab.server.protocol.state import SessionState; print('OK')"

# Убедиться что type checker теперь проверяет обращения:
grep -rn "mcp_manager\." src/codelab/ --include="*.py"
# Для каждого — убедиться что вызываемый метод существует в MCPManager
```
