## Причина закрытия

Документ предлагает добавить проверку разрешений в `FileSystemHandler` на клиенте, но это **не требуется** согласно архитектуре ACP:

1. **Разрешения уже проверяются на сервере** — `PermissionManager` в `prompt.py:432-475` проверяет `session/request_permission` перед выполнением tool call.
2. **RPC отправляется только после `allow`** — `fs/write_text_file` RPC вызывается только если пользователь разрешил операцию на этапе tool call.
3. **Клиент не должен спрашивать дважды** — добавление проверки в `FileSystemHandler` дублирует уже существующий флоу и нарушает архитектуру.

### Текущий корректный флоу:

```
LLM → Server: Tool Call (edit kind)
Server → User: session/request_permission
User → Server: allow_once
Server → Client: fs/write_text_file RPC (БЕЗ ПРОВЕРКИ — уже разрешено)
Client → Client: FileSystemHandler.execute()
Client → Server: {result: null}
```

### Альтернатива:

Если в будущем потребуется **дополнительная клиентская проверка** (например, локальные политики безопасности), это должно быть реализовано через отдельный `ClientPermissionChecker`, а не через дублирование `session/request_permission`.

В `FileSystemHandler` операции записи выполняются без запроса разрешения у пользователя:

```python
# file_system_handler.py:113
# TODO: Запросить разрешение у пользователя (Фаза 5)

# file_system_handler.py:147
# TODO: Запросить разрешение у пользователя (Фаза 5)
async def handle_write(self, request: dict) -> dict:
    # Запись происходит немедленно, без подтверждения
    await self._executor.write_text_file(path, content)
```

Агент может перезаписать произвольные файлы внутри рабочей директории без ведома пользователя. Согласно архитектуре ACP, операции записи и выполнения должны проходить через `PermissionHandler`.

---

## Решение

### Шаг 1 — Понять текущий flow разрешений

Разрешения в ACP работают через `PermissionHandler`:
1. Сервер отправляет клиенту `session/request_permission` с описанием операции.
2. `PermissionHandler` на клиенте отображает пользователю запрос через `PermissionViewModel`.
3. Пользователь выбирает `allow_once` / `allow_always` / `reject_once` / `reject_always`.
4. Клиент отвечает серверу — операция выполняется или отклоняется.

Однако запросы `fs/writeTextFile` идут напрямую от сервера к `FileSystemHandler`, минуя этот flow. Нужно добавить проверку разрешения **перед** выполнением операции записи.

### Шаг 3 — Добавить проверку перед записью

```python
async def handle_write_text_file(self, request: dict) -> dict:
    """Обработка запроса fs/writeTextFile с запросом разрешения."""
    params = request.get("params", {})
    path = params.get("path", "")
    content = params.get("content", "")
    request_id = request.get("id")

    self._logger.info("fs_write_requested", path=path)

    # Запросить разрешение у пользователя перед записью
    if self._permission_handler is not None:
        permission_granted = await self._permission_handler.request_permission(
            request_id=request_id,
            operation_kind="edit",
            operation_title=f"Write file: {path}",
            operation_description=f"The agent wants to write to: {path}",
        )
        if not permission_granted:
            self._logger.info("fs_write_rejected_by_user", path=path)
            return {
                "error": {
                    "code": -32001,
                    "message": "Permission denied by user",
                }
            }

    # Разрешение получено — выполнить запись
    try:
        await self._executor.write_text_file(path, content)
        self._logger.info("fs_write_completed", path=path)
        return {"result": {"success": True}}
    except Exception as e:
        self._logger.error("fs_write_failed", path=path, error=str(e))
        return {
            "error": {
                "code": -32603,
                "message": f"Write failed: {e}",
            }
        }
```

### Шаг 5 — Добавить метод `request_permission` в `PermissionHandler`

Если метод ещё не существует, добавить в `PermissionHandler`:

```python
async def request_permission(
    self,
    request_id: str | int | None,
    operation_kind: str,
    operation_title: str,
    operation_description: str,
) -> bool:
    """Запросить разрешение у пользователя через UI.

    Returns:
        True если пользователь разрешил операцию, False если отклонил.
    """
    # Отправить запрос через transport
    permission_request = {
        "jsonrpc": "2.0",
        "method": "session/request_permission",
        "id": request_id,
        "params": {
            "kind": operation_kind,
            "title": operation_title,
            "description": operation_description,
            "options": [
                {"optionId": "allow_once", "name": "Allow once", "kind": "allow_once"},
                {"optionId": "allow_always", "name": "Allow always", "kind": "allow_always"},
                {"optionId": "reject_once", "name": "Reject", "kind": "reject_once"},
            ],
        },
    }

    # Уведомить PermissionViewModel через EventBus
    if self._event_bus:
        await self._event_bus.publish("permission_requested", permission_request)

    # Ожидать ответа пользователя (через asyncio.Future)
    future: asyncio.Future[bool] = asyncio.get_event_loop().create_future()
    self._pending_requests[request_id] = future

    try:
        result = await asyncio.wait_for(future, timeout=300.0)  # 5 минут таймаут
        return result
    except asyncio.TimeoutError:
        self._logger.warning("permission_request_timeout", request_id=request_id)
        return False
    finally:
        self._pending_requests.pop(request_id, None)
```

@pytest.mark.asyncio
async def test_write_requires_permission(tmp_path):
    from unittest.mock import AsyncMock, MagicMock

    executor = FileSystemExecutor(base_path=tmp_path)
    permission_handler = MagicMock()
    permission_handler.request_permission = AsyncMock(return_value=False)

    handler = FileSystemHandler(executor=executor, permission_handler=permission_handler)

    result = await handler.handle_write_text_file({
        "id": "req-1",
        "params": {"path": "test.txt", "content": "hello"},
    })

    # Файл не должен быть создан
    assert not (tmp_path / "test.txt").exists()
    assert "error" in result
    assert result["error"]["code"] == -32001
