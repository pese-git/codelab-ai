# 1.7 — Ограничение размера входящих WebSocket сообщений

**Приоритет:** 🟡 Средний  
**Оценка:** 1 час  
**Файл:** `src/codelab/server/http_server.py`

---

## Проблема

В `ACPHttpServer` WebSocket принимается без ограничения размера сообщений:

```python
ws = web.WebSocketResponse()
await ws.prepare(request)
```

Клиент может отправить одно сообщение размером в несколько гигабайт. Сервер попытается загрузить его полностью в память перед обработкой, что приведёт к OOM или резкому росту потребления памяти. Это классический вектор DoS-атаки.

Для ACP-протокола типичный размер сообщения: несколько килобайт (текст промпта + метаданные). Даже очень большой промпт с вложениями редко превышает 1–2 МБ.

---

## Решение

### Шаг 1 — Добавить `max_msg_size` при создании WebSocket

```python
# БЫЛО:
async def _websocket_handler(self, request: web.Request) -> web.WebSocketResponse:
    ws = web.WebSocketResponse()
    await ws.prepare(request)
```

```python
# СТАЛО:
# Константы вынести на уровень модуля или конфига
WS_MAX_MSG_SIZE = 4 * 1024 * 1024  # 4 МБ — с запасом для больших промптов с контекстом

async def _websocket_handler(self, request: web.Request) -> web.WebSocketResponse:
    ws = web.WebSocketResponse(
        max_msg_size=WS_MAX_MSG_SIZE,
        heartbeat=30.0,   # Пинг каждые 30 секунд для обнаружения зависших соединений
    )
    await ws.prepare(request)
```

### Шаг 2 — Добавить конфигурацию через `AppConfig`

Чтобы лимит можно было менять без изменения кода:

```python
# config.py — добавить поле:
@dataclass
class AppConfig:
    ...
    ws_max_msg_size: int = 4 * 1024 * 1024  # 4 MB
    ws_heartbeat_interval: float = 30.0
```

```python
# http_server.py — использовать из конфига:
ws = web.WebSocketResponse(
    max_msg_size=self.config.ws_max_msg_size,
    heartbeat=self.config.ws_heartbeat_interval,
)
```

### Шаг 3 — Обработать ошибку превышения лимита

Когда сообщение превышает лимит, `aiohttp` закрывает соединение с кодом 1009 (Message Too Big). Нужно убедиться, что это логируется:

```python
async for msg in ws:
    if msg.type == WSMsgType.TEXT:
        await self._handle_message(ws, protocol, msg.data)
    elif msg.type == WSMsgType.ERROR:
        # Сюда попадают ошибки включая превышение лимита
        logger.warning(
            "ws_error",
            exception=str(ws.exception()),
            peer=request.remote,
        )
        break
    elif msg.type == WSMsgType.CLOSE:
        logger.info("ws_closed", peer=request.remote, code=msg.data)
        break
```

### Шаг 4 — Добавить лимит на размер одного JSON поля

Дополнительная защита — ограничить размер текста промпта на уровне протокола:

```python
# В handlers/prompt.py — при получении session/prompt:
MAX_PROMPT_TEXT_LENGTH = 100_000  # 100 тыс. символов

def validate_prompt_text(text: str) -> str:
    if len(text) > MAX_PROMPT_TEXT_LENGTH:
        raise ValueError(
            f"Prompt text too long: {len(text)} chars (max {MAX_PROMPT_TEXT_LENGTH})"
        )
    return text
```

## Проверка результата

```python
# Тест: отправка сообщения, превышающего лимит
@pytest.mark.asyncio
async def test_oversized_message_rejected(aiohttp_client, app):
    client = await aiohttp_client(app)
    ws = await client.ws_connect("/acp/ws")

    # Отправить сообщение размером 5 МБ (больше лимита 4 МБ)
    oversized = "x" * (5 * 1024 * 1024)
    await ws.send_str(oversized)

    # Соединение должно быть закрыто с кодом 1009
    msg = await ws.receive()
    assert msg.type in (aiohttp.WSMsgType.CLOSE, aiohttp.WSMsgType.ERROR)
```
