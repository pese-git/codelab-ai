# 2.2 — Устранение дублирования пакетов `content`

**Приоритет:** 🟠 Высокий  
**Оценка:** 1 день  
**Файлы:** `codelab/src/codelab/server/protocol/content/`, `codelab/src/codelab/shared/content/`

---

## Проблема

В проекте два пакета с пересекающимися файлами:

| `server/protocol/content/` | `shared/content/` | Статус |
|---|---|---|
| `base.py` | `base.py` | 🔴 Дублирование (в server меньше докстрингов) |
| `audio.py` | `audio.py` | 🔴 Дублирование (в server меньше докстрингов) |
| `embedded.py` | `embedded.py` | 🔴 Дублирование (в server импортирует из локального `base.py`) |
| `image.py` | `image.py` | 🔴 Дублирование (в server меньше докстрингов) |
| `text.py` | `text.py` | 🔴 Дублирование (в server меньше докстрингов) |
| `resource_link.py` | `resource_link.py` | 🔴 Дублирование (в server меньше докстрингов) |
| `extractor.py` | *(нет)* | ✅ Только в server |
| `formatter.py` | *(нет)* | ✅ Только в server (импортирует из локального `extractor.py`) |
| `validator.py` | *(нет)* | ✅ Только в server |

При изменении бизнес-логики в одном месте второе место останется устаревшим.

**Текущее состояние импортов:**
- `tests/server/test_content_*.py` — импортируют из `codelab.server.protocol.content`
- `tests/client/test_content_*.py` — импортируют из `codelab.shared.content`
- `codelab/shared/content/embedded.py` — импортирует из `codelab.shared.content.base` ✅
- `codelab/server/protocol/content/embedded.py` — импортирует из `codelab.server.protocol.content.base` 🔴

---

## Стратегия миграции

1. Оставить дублирующиеся модели в `shared/content/` — это единственный источник правды (там более полные докстринги).
2. Обновить `server/protocol/content/embedded.py` — заменить импорт с локального `base.py` на `shared/content/base.py`.
3. Удалить дублирующиеся файлы из `server/protocol/content/`.
4. Обновить `server/protocol/content/__init__.py` — реэкспортировать из shared.
5. Обновить `formatter.py` — импорты остаются локальными (зависит от `extractor.py`).
6. Обновить все импорты в `tests/server/` и других местах.

---

## Шаг 1 — Сравнить содержимое дублирующихся файлов

Перед удалением убедиться, что `server/protocol/content/` не содержит расширений, которых нет в `shared/content/`:

```bash
# Сравнить каждую пару:
diff codelab/src/codelab/server/protocol/content/base.py codelab/src/codelab/shared/content/base.py
diff codelab/src/codelab/server/protocol/content/audio.py codelab/src/codelab/shared/content/audio.py
diff codelab/src/codelab/server/protocol/content/text.py codelab/src/codelab/shared/content/text.py
# ... и т.д.
```

**Результат сравнения (текущее состояние):**
- `shared/` версии имеют более полные докстринги (примеры использования, Args/Returns/Raises)
- `server/` версии — минималистичные, без подробных описаний
- Функционально идентичны, `shared/` — более полная версия

Если в `server/protocol/content/base.py` есть что-то, чего нет в `shared/content/base.py` — добавить в `shared/content/base.py` перед удалением.

### Шаг 2 — Обновить `shared/content/__init__.py`

Убедиться, что `shared/content/` экспортирует всё необходимое:

```python
# codelab/src/codelab/shared/content/__init__.py — ТЕКУЩЕЕ СОСТОЯНИЕ:
from codelab.shared.content.audio import ALLOWED_AUDIO_MIME_TYPES, AudioContent
from codelab.shared.content.base import (
    BlobResource,
    ContentBlock,
    EmbeddedResource,
    TextResource,
)
from codelab.shared.content.embedded import EmbeddedResourceContent
from codelab.shared.content.image import ALLOWED_IMAGE_MIME_TYPES, ImageContent
from codelab.shared.content.resource_link import ResourceLinkContent
from codelab.shared.content.text import TextContent

__all__ = [
    # Базовые типы ресурсов
    "TextResource",
    "BlobResource",
    "EmbeddedResource",
    "ContentBlock",
    # Типы контента
    "TextContent",
    "ImageContent",
    "AudioContent",
    "EmbeddedResourceContent",
    "ResourceLinkContent",
    # Константы
    "ALLOWED_IMAGE_MIME_TYPES",
    "ALLOWED_AUDIO_MIME_TYPES",
]
```

**Примечание:** В коде используются `ContentBlock` и `EmbeddedResource` (не `ContentBase` и `ContentType` как могло быть в ранних версиях).

### Шаг 3 — Обновить `server/protocol/content/__init__.py` — оставить только специфичное

```python
# codelab/src/codelab/server/protocol/content/__init__.py — ПОСЛЕ рефакторинга:
"""Content extraction и validation для ACP протокола.

Модули для работы с content типами и extraction/validation.
Типы контента реэкспортируются из shared для обратной совместимости.
"""

# Реэкспортируем из shared для обратной совместимости внутренних импортов
from codelab.shared.content import (
    AudioContent,
    BlobResource,
    EmbeddedResourceContent,
    ImageContent,
    ResourceLinkContent,
    TextContent,

# Специфичные для сервера утилиты остаются здесь:

    # из shared (реэкспорт):
    "TextContent",
    "AudioContent",
    "ImageContent",
    "EmbeddedResourceContent",
    "ResourceLinkContent",
    "TextResource",
    "BlobResource",
    # специфичные для сервера:
    "ContentExtractor",
    "ExtractedContent",
    "ContentValidator",
    "ContentFormatter",
]
```

### Шаг 4 — Обновить `server/protocol/content/embedded.py` перед удалением

### Шаг 6 — Проверить `formatter.py` и `extractor.py`

`formatter.py` импортирует из локального `extractor.py` — это корректно, оба остаются в server:

```python
# codelab/src/codelab/server/protocol/content/formatter.py — БЕЗ ИЗМЕНЕНИЙ:

```

`extractor.py` и `validator.py` не импортируют типы контента напрямую — без изменений.

### Шаг 8 — Обновить тесты

Тесты дублируются аналогично модулям:

```bash
# Server тесты (импортируют из server/protocol/content):
ls codelab/tests/server/test_content_*.py

# test_content_formatting.py, test_content_extraction.py, test_content_image.py,
# test_content_integration.py, test_content_resource_link.py, test_content_text.py

# Client тесты (импортируют из shared/content):
ls codelab/tests/client/test_content_*.py
# test_content_audio.py, test_content_base.py, test_content_embedded.py,
# test_content_image.py, test_content_integration.py, test_content_resource_link.py,

```

**Варианты:**
1. **Объединить** — перенести в `tests/shared/test_content_*.py` и удалить дубликаты
2. **Оставить раздельно** — если тесты проверяют разное поведение (server vs client специфика)
3. **Оставить серверные тесты** — они импортируют через реэкспорт из `__init__.py` (Шаг 3)

Рекомендуется вариант 3: оставить серверные тесты как есть (они работают через реэкспорт), клиентские тесты уже используют `shared/content`.
