# 0.2 — Защита от повторной утечки секретов: pre-commit хуки

**Приоритет:** 🔴 НЕМЕДЛЕННО — сразу после выполнения 0.1  
**Оценка:** 1 час  
**Цель:** Не допустить повторного попадания секретов в репозиторий

---

## Проблема

Отзыв ключа и очистка истории — разовая операция. Без автоматической защиты та же ситуация повторится. Нужен механизм, который блокирует коммит при обнаружении потенциальных секретов.

---

## Решение: `pre-commit` + `detect-secrets`

### Шаг 1 — Установить `pre-commit`

```bash
pip install pre-commit
# или через uv (если используется):
uv add --dev pre-commit
```

### Шаг 2 — Создать конфигурацию

Создать файл `.pre-commit-config.yaml` в корне репозитория:

```yaml
# .pre-commit-config.yaml
repos:
  # Обнаружение секретов (API ключи, пароли, токены)
  - repo: https://github.com/Yelp/detect-secrets
    rev: v1.5.0
    hooks:
      - id: detect-secrets
        args: ["--baseline", ".secrets.baseline"]
        exclude: |
          (?x)^(
            .*\.example|
            .*\.md|
            tests/.*
          )$

  # Запрет коммита .env файлов
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.6.0
    hooks:
      - id: check-added-large-files
        args: ["--maxkb=500"]
      - id: detect-private-key
      - id: no-commit-to-branch
        args: ["--branch", "main", "--branch", "master"]

  # Дополнительно: trufflehog для глубокого сканирования
  - repo: https://github.com/trufflesecurity/trufflehog
    rev: v3.82.6
    hooks:
      - id: trufflehog
        name: TruffleHog
        entry: trufflehog git file://. --since-commit HEAD --only-verified --fail
        language: system
        pass_filenames: false
```

### Шаг 3 — Создать baseline для `detect-secrets`

Baseline фиксирует уже известные (намеренные) строки, похожие на секреты, чтобы они не блокировали коммиты:

```bash
# Создать baseline — сканирует весь текущий код:
detect-secrets scan > .secrets.baseline

# Просмотреть и проверить baseline:
cat .secrets.baseline

# Добавить baseline в репозиторий (это безопасно — содержит только хэши, не сами значения):
git add .secrets.baseline
```

### Шаг 4 — Установить хуки

```bash
pre-commit install
pre-commit install --hook-type commit-msg

# Проверить работу на всех существующих файлах:
pre-commit run --all-files
```

### Шаг 5 — Добавить в CI/CD

Чтобы хуки работали и на сервере (не только локально), добавить в pipeline:

**GitHub Actions** (`.github/workflows/security.yml`):
```yaml
name: Security Scan

on: [push, pull_request]

jobs:
  secrets-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Нужно для полного сканирования истории

      - name: Run detect-secrets
        run: |
          pip install detect-secrets
          detect-secrets scan --baseline .secrets.baseline
          detect-secrets audit .secrets.baseline

      - name: Run TruffleHog
        uses: trufflesecurity/trufflehog@main
        with:
          path: ./
          base: ${{ github.event.repository.default_branch }}
          head: HEAD
          extra_args: --only-verified
```

## Проверка результата

```bash
# Попытаться закоммитить файл с тестовым ключом:
echo 'API_KEY=sk-proj-testkey123456789012345678901234567890' > test_secret.txt
git add test_secret.txt
git commit -m "test"

# Ожидаемый вывод: хук заблокирует коммит с сообщением об обнаружении секрета
# Убрать тестовый файл:
rm test_secret.txt
git checkout .
```
