# 0.1 — Отзыв скомпрометированного API-ключа и очистка истории Git

**Приоритет:** 🔴 НЕМЕДЛЕННО — до следующего деплоя и до любых других задач  
**Оценка:** 1–2 часа  
**Файл-источник проблемы:** `acp-server/.env`

---

## Проблема

Файл `acp-server/.env` содержит реальный API-ключ OpenAI и был зафиксирован в Git:

```
ACP_LLM_API_KEY=sk-proj-****************************
ACP_LLM_BASE_URL=https://api-openai.st.by/v1
```

`.gitignore` защищает только **незакоммиченные** файлы. Если файл уже попал в историю, он доступен любому, у кого есть доступ к репозиторию — в том числе через `git log`, `git show`, `git clone`.

---

## Шаг 1 — Немедленно отозвать ключ в панели OpenAI

1. Открыть [https://platform.openai.com/api-keys](https://platform.openai.com/api-keys).
2. Найти ключ `sk-proj-****************************`.
3. Нажать **Revoke** (Отозвать).
4. Убедиться, что статус сменился на `Revoked`.

> Даже если планируется дальнейшее использование этого ключа — сначала отозвать, потом создать новый. Скомпрометированный ключ не восстанавливается в безопасное состояние.

---

## Шаг 2 — Удалить файл из истории Git

### Вариант A: `git-filter-repo` (рекомендуется)

```bash
# Установить инструмент (если не установлен):
pip install git-filter-repo

# Перейти в корень репозитория:
cd /path/to/acp-workspace

# Удалить файл из всей истории:
git filter-repo --path acp-server/.env --invert-paths --force

# Принудительно запушить очищенную историю:
git push origin --force --all
git push origin --force --tags
```

### Вариант B: `git filter-branch` (встроенный, но медленнее)

```bash
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch acp-server/.env' \
  --prune-empty --tag-name-filter cat -- --all

git push origin --force --all
git push origin --force --tags
```

### После очистки — сообщить всей команде

Все разработчики должны **удалить локальные клоны** и сделать свежий `git clone`. Принудительный push перезаписывает историю — старые клоны станут несовместимы.

```bash
# Каждый разработчик выполняет:
cd ..
rm -rf acp-workspace
git clone <repo-url> acp-workspace
```

---

## Шаг 3 — Добавить `acp-server/.env` в `.gitignore`

Открыть `.gitignore` в корне репозитория и убедиться, что присутствует явная строка:

```gitignore
# Секреты — никогда не коммитить
acp-server/.env
codelab/.env
.env
**/.env
```

Проверить, что файл действительно игнорируется:

```bash
git check-ignore -v acp-server/.env
# Ожидаемый вывод: .gitignore:N:acp-server/.env  acp-server/.env
```

---

## Шаг 4 — Создать новый ключ и сохранить безопасно

1. Создать новый API-ключ в панели OpenAI.
2. Сохранить в менеджере паролей (1Password, Bitwarden, etc.) или в secrets-хранилище CI/CD (GitHub Secrets, GitLab CI Variables).
3. Создать `acp-server/.env` локально из шаблона:

```bash
cp acp-server/.env.example acp-server/.env
# Открыть и вставить новый ключ
```

4. Убедиться, что новый `.env` не попадёт в коммит:

```bash
git status
# Файл acp-server/.env не должен появляться в списке
```

---

## Шаг 5 — Проверить, не попали ли секреты ещё куда-нибудь

```bash
# Поиск по всей истории Git:
git log --all -p | grep "sk-proj"

# Поиск в текущем коде:
grep -r "sk-proj\|sk-[a-zA-Z0-9]\{48\}" --include="*.py" --include="*.env*" --include="*.yml" .
```

Если ключ найден ещё в каких-либо файлах — повторить процедуру удаления для каждого.

---

## Проверка результата

```bash
# После очистки — убедиться, что ключ не найден в истории:
git log --all --oneline | head -20   # просмотр коммитов
git show HEAD:acp-server/.env 2>&1   # должно быть: fatal: Path 'acp-server/.env' ...

# Клонировать репозиторий заново и проверить:
cd /tmp && git clone <repo-url> check-repo
grep -r "sk-proj" check-repo/
# Результат должен быть пустым
```
