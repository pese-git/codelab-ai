# Быстрый старт

> Запуск CodeLab и выполнение первого запроса за 5 минут.

## Обзор

В этом руководстве вы:
1. Запустите CodeLab в локальном режиме
2. Выполните первый prompt
3. Увидите агента в действии

## Шаг 1: Запуск CodeLab

Откройте терминал и запустите:

```bash
cd acp-protocol/codelab

# Локальный режим (stdio транспорт, сервер как subprocess)
uv run codelab
```

Или запустите сервер отдельно и подключитесь клиентом:

```bash
# Терминал 1: сервер
uv run codelab serve --port 8765

# Терминал 2: клиент
uv run codelab connect --port 8765
```

Вы увидите:
```
INFO     Server starting on 127.0.0.1:8765
INFO     Web UI available at http://127.0.0.1:8765/
INFO     WebSocket endpoint: ws://127.0.0.1:8765/ws
INFO     Using LLM provider: mock
```

> **Совет:** Для работы с реальным LLM добавьте переменные окружения:
> ```bash
> OPENAI_API_KEY=sk-... CODELAB_LLM_PROVIDER=openai uv run codelab serve
> ```

## Шаг 2: Первый prompt

Введите простой запрос:

```
> Создай файл hello.py с функцией приветствия
```

Агент ответит планом действий:

```
📋 План:
1. Создать файл hello.py
2. Добавить функцию greet()
3. Добавить пример использования

🔧 Выполняю...
```

## Шаг 3: Система разрешений

При создании файла появится запрос разрешения:

```
┌─────────────────────────────────────────────────────────────┐
│  🔐 Запрос разрешения                                       │
│                                                             │
│  Агент хочет: Создать файл                                  │
│  Путь: hello.py                                             │
│                                                             │
│  [Allow]  [Allow All]  [Deny]                               │
└─────────────────────────────────────────────────────────────┘
```

- **Allow** — разрешить эту операцию
- **Allow All** — разрешить все подобные операции в сессии
- **Deny** — отклонить

Нажмите **Allow** или используйте клавишу `y`.

## Шаг 4: Результат

После подтверждения агент создаст файл:

```python
# hello.py

def greet(name: str) -> str:
    """Возвращает приветственное сообщение.
    
    Args:
        name: Имя для приветствия
        
    Returns:
        Строка с приветствием
    """
    return f"Привет, {name}!"


if __name__ == "__main__":
    print(greet("Мир"))
```

Сообщение об успехе:

```
✅ Файл hello.py создан успешно
```

## Альтернативные режимы запуска

### Web UI

Вместо TUI можно использовать браузер:

1. Запустите `codelab serve`
2. Откройте http://127.0.0.1:8765/ в браузере

### stdio транспорт (для IDE plugins)

```bash
# Сервер в stdio режиме
codelab serve --stdio

# Клиент запускает агент как subprocess
codelab connect --stdio --cwd /project
```

## Горячие клавиши TUI

| Клавиша | Действие |
|---------|----------|
| `Enter` | Отправить prompt |
| `Ctrl+C` | Отмена текущей операции |
| `Tab` | Переключение между панелями |
| `q` | Выход |
| `?` | Справка |

## Подключение MCP серверов

CodeLab поддерживает MCP (Model Context Protocol) серверы для расширения возможностей агента.

### Через TOML конфигурацию

Добавьте в `codelab.toml`:

```toml
[[mcp.servers]]
name = "filesystem"
type = "stdio"
command = "npx"
args = ["-y", "@modelcontextprotocol/server-filesystem", "/project"]
```

### Через сессию

При создании сессии укажите MCP серверы:

```json
{
  "method": "session/new",
  "params": {
    "mcpServers": [
      {
        "name": "filesystem",
        "type": "stdio",
        "command": "npx",
        "args": ["-y", "@modelcontextprotocol/server-filesystem", "/project"]
      }
    ]
  }
}
```

### Пример использования MCP

После подключения MCP сервера агент может использовать новые инструменты:

```
> Прочитай файл README.md через MCP filesystem

🔧 [MCP:filesystem] read_file: README.md
   Status: ✅ Completed
   ─────────────────────────
   # My Project
   ...
```

## Решение проблем

### Клиент не подключается

```bash
# Проверьте, что сервер запущен
curl http://127.0.0.1:8765/

# Проверьте порт
lsof -i :8765
```

### Mock-провайдер даёт странные ответы

Это нормально — mock предназначен для тестирования. Для реальной работы используйте OpenAI:

```bash
OPENAI_API_KEY=sk-... CODELAB_LLM_PROVIDER=openai uv run codelab serve
```

### TUI отображается некорректно

Убедитесь, что терминал поддерживает Unicode и 256 цветов:

```bash
# Проверка
echo $TERM
# Должно быть: xterm-256color или подобное
```
