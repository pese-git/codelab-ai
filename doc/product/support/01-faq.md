# FAQ — Часто задаваемые вопросы

## Общие вопросы

### Что такое CodeLab?

CodeLab — это AI-ассистент для программирования, реализующий Agent Client Protocol (ACP). Он предоставляет интеллектуального агента, который может работать с вашим кодом: читать файлы, выполнять команды терминала, создавать и редактировать код.

### Какие LLM провайдеры поддерживаются?

CodeLab поддерживает:
- **OpenAI** — GPT-4, GPT-4o и другие модели
- **Anthropic** — Claude 3 (Opus, Sonnet, Haiku)
- **OpenAI-совместимые API** — OpenRouter, Azure OpenAI, vLLM, Ollama

### Нужен ли API ключ?

Да, для работы с реальными LLM моделями необходим API ключ соответствующего провайдера. Для тестирования можно использовать `CODELAB_LLM_PROVIDER=mock`.

### Где хранятся мои сессии?

Сессии сохраняются в `~/.codelab/data/sessions/` в формате JSON. Вы можете загружать ранее созданные сессии и продолжать работу.

## Установка и настройка

### Как установить CodeLab?

```bash
# Клонирование репозитория
git clone https://github.com/pese-git/codelab-ai.git
cd codelab

# Установка зависимостей
uv sync

# Запуск
uv run codelab
```

### Как настроить API ключ?

Создайте файл `.env` в директории проекта или `~/.codelab/config/.env`:

```env
CODELAB_LLM_PROVIDER=openai
CODELAB_LLM_API_KEY=sk-your-key-here
```

### Как изменить порт сервера?

Через переменную окружения:
```bash
export CODELAB_PORT=4096
codelab serve
```

Или через параметр командной строки:
```bash
codelab serve --port 4096
```

## Работа с приложением

### Как создать новую сессию?

- Нажмите `Ctrl+N` в TUI клиенте
- Или используйте кнопку "New Session" в интерфейсе

### Как отменить выполняемый запрос?

Нажмите `Ctrl+C` для отмены текущего prompt turn.

### Как посмотреть вывод терминала?

Нажмите `Ctrl+T` для открытия окна вывода терминала.

### Как работает система разрешений?

Агент запрашивает разрешение на выполнение потенциально опасных операций:
- Чтение файлов
- Запись файлов
- Выполнение команд терминала

Вы можете разрешить операцию однократно или запомнить выбор для данной сессии.

## Проблемы и решения

### Приложение не запускается

1. Проверьте установку зависимостей: `uv sync`
2. Убедитесь, что Python 3.12+ установлен
3. Проверьте логи в `~/.codelab/logs/codelab.log`

### Не работает LLM

1. Проверьте правильность API ключа
2. Проверьте сетевое подключение
3. Попробуйте mock провайдер для тестирования

### Сессия не сохраняется

Убедитесь, что директория `~/.codelab/data/sessions/` существует и доступна для записи.

## Разработка

### Как интегрировать с IDE?

Используйте stdio транспорт для интеграции с Zed IDE и другими редакторами:

```bash
codelab serve --stdio
```

Подробнее: [Интеграция с Zed IDE](../user-guide/10-zed-ide-integration.md)

### Как подключить MCP серверы?

MCP серверы указываются при создании сессии через параметр `mcpServers`. CodeLab автоматически подключится к указанным MCP серверам и добавит их инструменты.

### Что такое Dishka DI?

Dishka — это DI контейнер, используемый CodeLab для управления зависимостями. На сервере используются два скоупа: APP (синглтоны) и REQUEST (per-connection).

```bash
cd codelab
uv run python -m pytest
```

### Как запустить линтер?

```bash
cd codelab
uv run ruff check .
```

### Как включить отладочные логи?

```bash
export CODELAB_LOG_LEVEL=DEBUG
codelab -v
```
