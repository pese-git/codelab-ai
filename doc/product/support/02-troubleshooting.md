# Решение проблем

Руководство по устранению распространённых проблем CodeLab.

## Проблемы запуска

### Ошибка "Module not found"

**Симптом:** При запуске появляется ошибка импорта модуля.

**Решение:**
```bash
cd codelab
uv sync --reinstall
```

### Ошибка "Port already in use"

**Симптом:** Сервер не запускается, порт занят.

**Решение:**
1. Используйте другой порт:
   ```bash
   codelab serve --port 9000
   ```

2. Найдите и завершите процесс, использующий порт:
   ```bash
   # macOS/Linux
   lsof -i :8765
   kill -9 <PID>
   
   # Windows
   netstat -ano | findstr :8765
   taskkill /PID <PID> /F
   ```

### Ошибка "Permission denied"

**Симптом:** Не удаётся создать файлы в домашней директории.

**Решение:**
```bash
# Проверить права
ls -la ~/.codelab/

# Исправить права
chmod -R 755 ~/.codelab/
```

## Проблемы с LLM

### Ошибка "Invalid API Key"

**Симптом:** LLM запросы завершаются с ошибкой аутентификации.

**Проверка:**
1. Убедитесь, что ключ указан в правильной переменной окружения (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, и т.д.)
2. Проверьте, что ключ активен в личном кабинете провайдера
3. Убедитесь, что выбран правильный провайдер

```bash
# Проверить текущие настройки
cat .env
```

### Ошибка "Rate limit exceeded"

**Симптом:** Запросы отклоняются из-за превышения лимитов.

**Решение:**
1. Подождите несколько минут
2. Настройте fallback цепочку для автоматического переключения на резервный провайдер
3. Проверьте тарифный план API провайдера

```toml
# codelab.toml — fallback при rate limit
[llm.fallback]
enabled = true
order = ["openai", "openrouter", "ollama"]
retry_on = ["rate_limit", "timeout"]
```

### Ошибка "Model not found"

**Симптом:** Указанная модель не найдена.

**Решение:**
Проверьте название модели в формате `"provider/model"`:

```env
# Правильный формат
CODELAB_LLM_MODEL=openai/gpt-4o
CODELAB_LLM_MODEL=anthropic/claude-sonnet-4
CODELAB_LLM_MODEL=ollama/llama3.1:70b
```

### Ошибка "Provider not found"

**Симптом:** Провайдер не найден в Registry.

**Решение:**
Проверьте ID провайдера. Доступные: `openai`, `anthropic`, `openrouter`, `zen`, `go`, `ollama`, `lmstudio`, `mock`.

### Fallback не срабатывает

**Симптом:** При ошибке основного провайдера fallback не переключается на резервный.

**Решение:**
1. Проверьте `CODELAB_FALLBACK_ENABLED=true`
2. Убедитесь что резервные провайдеры зарегистрированы
3. Проверьте `retry_on` — ошибка должна быть retryable (`rate_limit`, `timeout`, `internal_error`, `service_unavailable`)

### Нет ответа от LLM

**Симптом:** Запрос отправлен, но ответа нет.

**Проверка:**
1. Включите debug логи: `CODELAB_LOG_LEVEL=DEBUG`
2. Проверьте сетевое подключение
3. Убедитесь, что BASE_URL указан правильно (если используется)
4. Проверьте что Ollama/LMStudio запущен (для локальных провайдеров)

## Проблемы с TUI

### Искажённое отображение

**Симптом:** Символы отображаются неправильно.

**Решение:**
1. Используйте терминал с поддержкой Unicode
2. Установите шрифт с поддержкой символов (Nerd Fonts)
3. Проверьте настройки локали:
   ```bash
   export LANG=en_US.UTF-8
   ```

### Не работают горячие клавиши

**Симптом:** Комбинации клавиш не срабатывают.

**Решение:**
1. Убедитесь, что фокус на нужном элементе
2. Проверьте, не перехватывает ли терминал сочетания
3. Попробуйте альтернативные сочетания (нажмите `?` для списка)

### TUI зависает

**Симптом:** Интерфейс не отвечает.

**Решение:**
1. Нажмите `Ctrl+C` для отмены текущей операции
2. Если не помогает, нажмите `Ctrl+Q` для выхода
3. В крайнем случае завершите процесс:
   ```bash
   pkill -f codelab
   ```

## Проблемы с сессиями

### Сессия не загружается

**Симптом:** Ранее созданная сессия не появляется в списке.

**Проверка:**
1. Убедитесь, что файл сессии существует:
   ```bash
   ls -la ~/.codelab/data/sessions/
   ```

2. Проверьте корректность JSON:
   ```bash
   python -m json.tool ~/.codelab/data/sessions/<session-id>.json
   ```

### Потеряна история сессии

**Симптом:** История сообщений пуста после перезапуска.

**Причины:**
- Сессия не была сохранена (аварийное завершение)
- Повреждён файл сессии

**Решение:**
Регулярно сохраняйте важные сессии. Используйте систему контроля версий для критичных проектов.

## Проблемы с инструментами

### Файловые операции отклоняются

**Симптом:** Агент не может прочитать/записать файлы.

**Проверка:**
1. Убедитесь, что рабочая директория указана правильно
2. Проверьте права доступа к файлам
3. Разрешите операцию в диалоге разрешений

### Команды терминала не выполняются

**Симптом:** Терминальные команды возвращают ошибки.

**Проверка:**
1. Убедитесь, что команда доступна в PATH
2. Проверьте рабочую директорию сессии
3. Посмотрите вывод терминала (`Ctrl+\``)

## Проблемы с stdio режимом

### Агент не запускается

**Симптом:** `codelab connect --stdio` возвращает ошибку подключения.

**Проверка:**
1. Проверьте, что `codelab serve --stdio` работает вручную
2. Посмотрите stderr агента
3. Проверьте логи в `~/.codelab/logs/codelab.log`

### stdio процесс завершается неожиданно

**Симптом:** Агент завершается во время работы.

**Проверка:**
1. Проверьте логи процесса
2. Убедитесь, что хватает памяти
3. Проверьте, что LLM провайдер настроен корректно

## Проблемы с MCP

### MCP сервер не запускается

**Симптом:** Ошибка при создании сессии с MCP сервером.

**Проверка:**
1. Команда запуска корректна:
   ```bash
   npx -y @modelcontextprotocol/server-filesystem /project
   ```
2. Зависимости установлены:
   ```bash
   npm list -g @modelcontextprotocol/server-filesystem
   ```
3. Переменные окружения установлены:
   ```bash
   echo $GITHUB_TOKEN
   ```

**Решение:**
```bash
# Установка MCP сервера
npm install -g @modelcontextprotocol/server-filesystem

# Проверка запуска
npx @modelcontextprotocol/server-filesystem /project --stdio
```

### MCP сервер отключился

**Симптом:** Инструменты MCP сервера недоступны.

**Автоматическое восстановление:**
1. CodeLab обнаруживает отключение через health check (каждые 60с)
2. Запускает переподключение с exponential backoff
3. Обновляет список инструментов после подключения

**Ручная проверка:**
```bash
# Проверка логов
cat ~/.codelab/logs/codelab.log | grep "mcp"
```

### Инструменты MCP не видны агенту

**Симптом:** Агент не использует MCP инструменты.

**Проверка:**
1. Сервер подключён:
   ```
   MCP серверы: filesystem (5 tools), github (12 tools)
   ```
2. Инструменты зарегистрированы в ToolRegistry
3. Capabilities клиента поддерживают MCP инструменты

**Решение:**
- Пересоздайте сессию с MCP серверами
- Проверьте логи на ошибки инициализации

### Timeout при вызове MCP инструмента

**Симптом:** `MCP tool execution error: timeout`

**Причины:**
- MCP сервер медленно отвечает
- Сетевые проблемы (для HTTP/SSE)
- Сервер завис

**Решение:**
1. Увеличьте timeout в конфигурации
2. Проверьте доступность сервера
3. Перезапустите MCP сервер

### Ошибка переподключения MCP

**Симптом:** MCP сервер не восстанавливает соединение после отключения.

**Проверка:**
1. Количество попыток не превышено (по умолчанию 5)
2. Сервер доступен
3. Логи показывают причины失败

**Решение:**
```toml
# Увеличьте retry параметры
[[mcp.servers]]
name = "my-server"
max_retries = 10
initial_delay = 2.0
max_delay = 60.0
```

### Stdio MCP сервер падает

**Симптом:** MCP сервер завершается с ошибкой.

**Проверка:**
```bash
# Запуск вручную для проверки
npx @modelcontextprotocol/server-filesystem /project --stdio

# Проверка stderr
cat ~/.codelab/logs/codelab.log | grep "stderr"
```

### HTTP MCP сервер недоступен

**Симптом:** Ошибка подключения к HTTP MCP серверу.

**Проверка:**
1. URL доступен:
   ```bash
   curl -v https://mcp.example.com/mcp
   ```
2. Headers корректны (auth token)
3. Сервер поддерживает MCP протокол

### Конфликт имён MCP инструментов

**Симптом:** Два MCP сервера предоставляют инструменты с одинаковыми именами.

**Решение:**
CodeLab использует namespaced имена `mcp:server_id:tool_name`, поэтому конфликтов не возникает. Убедитесь, что `name` у серверов уникальны.

## Логирование и диагностика

### Как включить подробные логи

```bash
# Через переменную
export CODELAB_LOG_LEVEL=DEBUG

# Через флаг CLI
codelab -v serve
```

### Где находятся логи

```bash
~/.codelab/logs/codelab.log
```

### Как отправить отчёт об ошибке

1. Соберите логи с уровнем DEBUG
2. Запишите шаги для воспроизведения
3. Укажите версию Python и ОС
4. Создайте issue в репозитории проекта

## Сброс настроек

### Полный сброс конфигурации

```bash
# Удалить домашнюю директорию
rm -rf ~/.codelab/

# Заново запустить (создастся новая)
codelab
```

**Внимание:** Это удалит все сессии и настройки!
