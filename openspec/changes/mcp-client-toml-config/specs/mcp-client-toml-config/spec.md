## ADDED Requirements

### Requirement: Клиент загружает MCP серверы из TOML конфигурации

Клиент ACP SHALL загружать конфигурацию MCP серверов из секции `[[mcp.servers]]` TOML файлов при старте приложения и передавать её серверу через ACP протокол при создании или загрузке сессии.

#### Scenario: Загрузка MCP серверов из одного TOML файла
- **WHEN** файл `codelab.toml` содержит секцию `[[mcp.servers]]` с валидной конфигурацией
- **THEN** клиент загружает все MCP серверы и возвращает список словарей в формате ACP протокола

#### Scenario: Загрузка из цепочки TOML файлов с override по name
- **WHEN** несколько TOML файлов содержат `[[mcp.servers]]` с серверами имеющими одинаковое поле `name`
- **THEN** серверы из файлов с высшим приоритетом переопределяют серверы с тем же `name` из файлов с низшим приоритетом

#### Scenario: Пустой список при отсутствии MCP секции
- **WHEN** ни один из TOML файлов не содержит секцию `[[mcp.servers]]`
- **THEN** клиент возвращает пустой список MCP серверов

### Requirement: Env var expansion в MCP конфигурации

Клиент SHALL раскрывать переменные окружения в формате `${VAR_NAME}` во всех строковых полях MCP конфигурации перед передачей серверу.

#### Scenario: Expansion в command поле
- **WHEN** поле `command` содержит `${NPM_PATH}`
- **THEN** клиент заменяет `${NPM_PATH}` на значение переменной окружения `NPM_PATH`

#### Scenario: Expansion в args массиве
- **WHEN** элемент массива `args` содержит `${MCP_SERVER_ARG}`
- **THEN** клиент заменяет `${MCP_SERVER_ARG}` на значение переменной окружения

#### Scenario: Expansion в headers value
- **WHEN** поле `headers[].value` содержит `Bearer ${GITHUB_TOKEN}`
- **THEN** клиент заменяет `${GITHUB_TOKEN}` на значение переменной окружения `GITHUB_TOKEN`

#### Scenario: Expansion в env value
- **WHEN** поле `env[].value` содержит `${MY_API_KEY}`
- **THEN** клиент заменяет `${MY_API_KEY}` на значение переменной окружения

#### Scenario: Expansion в url поле
- **WHEN** поле `url` содержит `${MCP_BASE_URL}/mcp`
- **THEN** клиент заменяет `${MCP_BASE_URL}` на значение переменной окружения

#### Scenario: Отсутствующая переменная окружения
- **WHEN** строка содержит `${UNDEFINED_VAR}` и переменная окружения не установлена
- **THEN** клиент заменяет `${UNDEFINED_VAR}` на пустую строку

### Requirement: Валидация MCP конфигурации

Клиент SHALL валидировать каждый MCP сервер и пропускать невалидные конфигурации с записью warning в лог.

#### Scenario: Пропуск сервера без name
- **WHEN** конфигурация MCP сервера не содержит поле `name`
- **THEN** клиент пропускает этот сервер и записывает warning в лог

#### Scenario: Пропуск stdio сервера без command
- **WHEN** сервер имеет `type = "stdio"` но не содержит поле `command`
- **THEN** клиент пропускает этот сервер и записывает warning в лог

#### Scenario: Пропуск http сервера без url
- **WHEN** сервер имеет `type = "http"` но не содержит поле `url`
- **THEN** клиент пропускает этот сервер и записывает warning в лог

#### Scenario: Валидный stdio сервер
- **WHEN** сервер имеет `name`, `type = "stdio"` и `command`
- **THEN** клиент включает этот сервер в результат

#### Scenario: Валидный http сервер
- **WHEN** сервер имеет `name`, `type = "http"` и `url`
- **THEN** клиент включает этот сервер в результат

### Requirement: Передача MCP серверов при создании сессии

Клиент SHALL передавать загруженные MCP серверы серверу в параметре `mcpServers` метода `session/new`.

#### Scenario: Создание сессии с MCP серверами
- **WHEN** клиент вызывает `session/new` и имеет загруженные MCP серверы из TOML
- **THEN** клиент включает `mcpServers` массив в параметры запроса

#### Scenario: Создание сессии без MCP серверов
- **WHEN** клиент вызывает `session/new` и не имеет загруженных MCP серверов
- **THEN** клиент включает пустой `mcpServers` массив в параметры запроса

### Requirement: Передача MCP серверов при загрузке сессии

Клиент SHALL передавать загруженные MCP серверы серверу в параметре `mcpServers` метода `session/load`.

#### Scenario: Загрузка сессии с MCP серверами
- **WHEN** клиент вызывает `session/load` и имеет загруженные MCP серверы из TOML
- **THEN** клиент включает `mcpServers` массив в параметры запроса

### Requirement: MCPConfigLoader независим от серверных модулей

Клиентский `MCPConfigLoader` SHALL быть независимым от серверных модулей и использовать только стандартную библиотеку Python для парсинга TOML.

#### Scenario: Импорт только стандартных библиотек
- **WHEN** клиент импортирует `MCPConfigLoader`
- **THEN** импортируются только модули из `codelab.client.*` и стандартной библиотеки

### Requirement: Поддерживаемые поля MCP сервера

Клиент SHALL поддерживать следующие поля в конфигурации MCP сервера: `name`, `type`, `command`, `args`, `env`, `url`, `headers`.

#### Scenario: Stdio сервер с env
- **WHEN** сервер имеет `type = "stdio"`, `command`, `args`, `env`
- **THEN** клиент включает все поля в результат

#### Scenario: HTTP сервер с headers
- **WHEN** сервер имеет `type = "http"`, `url`, `headers`
- **THEN** клиент включает все поля в результат
